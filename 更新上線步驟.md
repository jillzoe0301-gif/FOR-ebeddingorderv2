# V4.8 更新上線步驟

本版接續 V4.7，使用原本的 Supabase、GitHub、Vercel 專案。

## 不需要重新操作

- 不需要重新建立 Supabase Project
- 不需要重新執行 Supabase SQL
- 不需要重新設定 Vercel 環境變數
- 不需要建立新的 GitHub Repository

## GitHub 更新方式

1. 解壓縮 `FOR-e_Bedding_Order_V4.8_GitHub_Update_Patch.zip`。
2. 將解壓縮後的檔案放到 GitHub Codespaces 專案根目錄。
3. 覆蓋同名檔案。
4. 在終端機執行：

```bash
git add .
git status
git commit -m "Exclude bike and fan from purchase order V4.8"
git push origin main
```

5. 推送完成後，Vercel 會自動建立新部署。

## 上線後測試

建立一張包含以下品項的測試訂單：

- 塑膠床墊 1
- 電風扇 1
- 腳踏車 1

確認：

1. 訂購單下載只顯示塑膠床墊。
2. 簽收單顯示塑膠床墊、電風扇、腳踏車。
3. 腳踏車費用獨立顯示。
4. 寢具及用品費用與總金額計算正確。
