# V4.10 更新部署步驟

本版接續 V4.9.1，新增訂單公司抬頭與發票開立抬頭分開顯示。

## 不需要重新操作

- 不需要重新建立 Supabase Project
- 不需要重新執行 Supabase SQL
- 不需要重新設定 Vercel 環境變數
- 不需要建立新的 GitHub Repository

## GitHub 更新方式

1. 解壓縮 `FOR-e_Bedding_Order_V4.10_GitHub_Update_Patch.zip`。
2. 將解壓縮後的全部檔案放到 Codespaces 專案根目錄。
3. 覆蓋同名檔案。
4. 在終端機依序執行：

```bash
git add .
git status
git commit -m "Separate company and invoice titles V4.10"
git push origin main
```

5. GitHub 推送完成後，Vercel 會自動部署新的 commit。

## 上線後確認

### 新增訂購

選擇：

```text
公司別（訂單抬頭）：灃禾管顧
```

系統應自動顯示：

```text
發票開立抬頭：灃康人力
```

### 訂購單

最上方標題應顯示：

```text
灃禾管顧入境外國人寢具訂購單
```

訂購單資料列應另外顯示：

```text
公司別／訂單抬頭：灃禾管顧
發票開立抬頭：灃康人力
```

### LINE

LINE 訊息應包含：

```text
公司別（訂單抬頭）：灃禾管顧
發票開立抬頭：灃康人力
```

## 若 Vercel 沒有自動部署

進入：

```text
Vercel → 專案 → Deployments
```

確認最新部署使用的是剛推送的新 commit，不要重新部署舊版本 commit。
