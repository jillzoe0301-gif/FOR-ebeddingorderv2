# FOR-e 寢具訂購系統 V4.6 更新上線步驟

## 更新前

先在目前系統執行：

```text
資料設定 → 下載備份 JSON
```

這是預防性備份，更新程式不會主動刪除 Supabase 資料。

## 一、解壓縮

請先解壓縮本次 ZIP。

不要把 ZIP 本身直接放進 GitHub。請將解壓縮後資料夾內的檔案放到 Repository 根目錄，讓根目錄直接看得到：

```text
index.html
package.json
package-lock.json
vercel.json
.npmrc
supabase/
```

Mac 若看不到 `.npmrc`，按：

```text
Command + Shift + .
```

## 二、在 GitHub Codespaces 覆蓋檔案

將解壓縮後的檔案拖入目前 `FOR-ebeddingorderv2` Codespaces 根目錄，遇到同名檔案選擇覆蓋。

## 三、提交 GitHub

在 Codespaces 終端機執行：

```bash
git add .
git status
git commit -m "Add item order and special item types V4.6"
git push origin main
```

`git status` 應看到 `index.html`、`package.json`、`package-lock.json` 等檔案已進入 `Changes to be committed`。

## 四、Vercel 自動部署

推送成功後，Vercel 會自動建立新 Deployment。

正常建置紀錄應包含：

```text
Running "install" command: npm ci --no-audit --no-fund
Running "npm run build"
✓ built
Build Completed
```

不要重新部署舊 commit，應等待這次 V4.6 新 commit 自動部署。

## 五、Supabase 與環境變數

從 V4.5.1 更新時：

- 不需要重新執行 Supabase SQL。
- 不需要新增資料表。
- 不需要建立帳號。
- 不需要修改 `VITE_SUPABASE_URL`。
- 不需要修改 `VITE_SUPABASE_ANON_KEY`。

## 六、部署後確認

1. 開啟正式網址，右上版本應顯示 V4.6。
2. 進入「資料設定」，確認品項列表有：
   - 訂購類型
   - 上移
   - 下移
3. 將任一品項改為特殊訂購項目並儲存。
4. 開啟「寢具訂購」，確認該品項出現在特殊訂購區塊。
5. 確認特殊訂購項目沒有拿庫存、加買、退貨欄位。
6. 建立測試訂單後，確認訂購明細與 CSV 順序正確。
