# FOR-e 寢具訂購系統 V4.7 更新上線步驟

## 更新前

先在目前系統執行：

```text
資料設定 → 下載備份 JSON
```

更新程式不會主動刪除 Supabase 資料，此步驟屬於預防性備份。

## 一、解壓縮

請先解壓縮本次 ZIP。

不要把 ZIP 本身直接放進 GitHub。請將解壓縮後資料夾裡面的檔案放到 Repository 根目錄，讓根目錄直接看得到：

```text
index.html
package.json
package-lock.json
vercel.json
.npmrc
supabase/
```

Mac 若看不到 `.npmrc`，按：

```text
Command + Shift + .
```

## 二、覆蓋 GitHub Codespaces 檔案

將解壓縮後資料夾中的檔案拖入目前 `FOR-ebeddingorderv2` Codespaces 根目錄，遇到同名檔案時選擇覆蓋。

## 三、提交 GitHub

在 Codespaces 終端機依序執行：

```bash
git add .
git status
git commit -m "Add bulk workers and order status V4.7"
git push origin main
```

`git status` 應看到 `index.html`、`package.json`、`package-lock.json`、`README.md` 等檔案位於 `Changes to be committed`。

## 四、Vercel 自動部署

推送成功後，Vercel 會自動建立新的 Deployment。

正常建置紀錄應包含：

```text
Running "install" command: npm ci --no-audit --no-fund
Running "npm run build"
✓ built
Build Completed
```

請等待 V4.7 的新 commit 自動部署，不要重新部署舊 commit。

## 五、Supabase 與環境變數

從 V4.6 更新時：

- 不需要重新執行 Supabase SQL。
- 不需要新增資料表。
- 不需要建立帳號。
- 不需要修改 `VITE_SUPABASE_URL`。
- 不需要修改 `VITE_SUPABASE_ANON_KEY`。

## 六、部署後確認

1. 開啟正式網址，左上版本顯示 V4.7。
2. 「寢具訂購」清單應顯示「入境日」及「目前狀態」。
3. 點「＋新增訂購」。
4. 部門選一部，承辦應自動為貝貝。
5. 部門改二部，承辦應自動改為阿金。
6. 在多工人欄位貼上三個姓名，按「一鍵新增工人」。
7. 確認共同國籍及共同入境日已帶入每位工人。
8. 填寫第一位工人的品項後，按「第一位工人購買品項套用全部」。
9. 儲存測試訂單，確認清單顯示入境日及目前狀態。
