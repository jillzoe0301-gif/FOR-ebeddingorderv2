# FOR-e 寢具訂購系統 V4.8

Supabase 公開共用免登入版，使用 GitHub 版本控管並由 Vercel 自動部署。

## V4.8 更新內容

1. 「寢具訂購」清單新增入境日欄位。
2. 「寢具訂購」及「訂購紀錄」清單明確顯示目前狀態。
3. 訂單明細新增「目前狀態」，並改善多位工人的入境日／國籍摘要。
4. 新增訂購選擇「一部」時，承辦自動帶入「貝貝」。
5. 新增訂購選擇「二部」時，承辦自動帶入「阿金」。
6. 可在同一欄位一次輸入多位工人姓名，每行一位或使用逗號、頓號、分號分隔。
7. 共同國籍與共同入境日可在建立工人時直接帶入，也可一鍵套用全部工人。
8. 可將第一位工人的購買品項、數量、拿庫存、加買及備註一鍵套用到其餘工人。
9. 保留「基本寢具套用全部」及單筆新增工人功能。

## 架構

- 前端：Vite 單頁應用程式
- 資料庫：Supabase
- 程式碼：GitHub
- 部署：Vercel
- 登入／權限：無，開啟網址直接使用

## 環境變數

Vercel 與本機均需設定：

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

不得將 Supabase service role key 放在前端或 GitHub。

## 更新既有 V4.6

目前 V4.6 已可正常連線 Supabase 時：

- 不需要重新建立 Supabase Project。
- 不需要重新執行 SQL。
- 不需要修改 Vercel 環境變數。
- 只需以本版檔案覆蓋 GitHub Repository 根目錄並推送。
- 原有訂單、工人、品項、庫存及對帳資料會保留。

完整操作請看 `更新上線步驟.md`。


## V4.8 重點

- 腳踏車、電風扇保留於工人簽收單。
