# FOR-e 寢具訂購系統 V4.9

Supabase 公開共用免登入版，使用 GitHub 版本控管並由 Vercel 自動部署。

## V4.9 更新內容

1. 使用指定的 Order 圖示作為瀏覽器分頁、我的最愛與手機主畫面圖示。
2. 系統左上角 LOGO 同步更換為同一張圖。
3. 資料設定新增「收貨聯絡人」管理，可設定姓名、分機／電話、預設送貨地點、地址、備註及啟用狀態。
4. 新增訂購時，收貨聯絡人可直接從設定資料選用，並自動帶入對應送貨地點與地址。
5. 灃康人力的一部／二部仍依承辦自動帶入貝貝／阿金及其聯絡資料。
6. 資料設定依實際操作流程重新分為：部門與承辦、收貨聯絡人、公司與雇主、寢具廠商、品項與價格、備份與還原。
7. 各區塊提供獨立儲存按鈕、說明文字與快速導覽，降低誤操作。

## 架構

- 前端：Vite 單頁應用程式
- 資料庫：Supabase
- 程式碼：GitHub
- 部署：Vercel
- 登入／權限：無，開啟網址直接使用

## 環境變數

Vercel 與本機均需設定：

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

不得將 Supabase service role key 放在前端或 GitHub。

## 更新既有版本

目前 V4.8 已可正常連線 Supabase 時：

- 不需要重新建立 Supabase Project。
- 不需要重新執行 SQL。
- 不需要修改 Vercel 環境變數。
- 只需以本版檔案覆蓋 GitHub Repository 根目錄並推送。
- 原有訂單、品項、庫存、對帳及設定資料都會保留。
- 舊版沒有收貨聯絡人資料時，系統會自動建立貝貝、阿金、明明三筆預設資料。

完整操作請看 `更新上線步驟.md`。
